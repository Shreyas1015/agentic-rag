SEC-01Build Phases
Three Phases
Don't try to build everything at once. Phase 1 gets you a working Agentic RAG. Phase 2 makes it observable and trustworthy. Phase 3 upgrades the quality ceiling. Each phase ships something testable end-to-end.

SEC-02Timeline
Week-by-Week Roadmap
Rough weekly breakdown. Each week ends with something testable. Don't skip weeks — the foundations matter.

PHASE 1Week 3 — Query Pipeline + Agent LoopDays 15–21
  • Redis semantic cache — embed query, cosine similarity check, 0.88 threshold
  • LangGraph AgentState model + all 7 nodes (use Cohere rerank as placeholder for now)
  • Hybrid Qdrant search — dense + sparse prefetch, RRF fusion
  • Parent chunk fetch from PostgreSQL
  • SSE streaming output from /chat/stream
  • End of week: Full Q&A pipeline working end-to-end. Ask a question, get a cited answer.
PHASE 2Week 4 — Observability + Reranker SwapDays 22–28
  • Add Langfuse to docker-compose.yml, configure self-hosted
  • Add @observe() decorators to every LangGraph node
  • Swap Cohere rerank → BGE-Reranker-v2-M3 local inference
  • Replace keyword classifier → Gemini Flash JSON classifier
  • Build 50-question eval dataset from your test documents
  • End of week: Every query has a Langfuse trace. BGE reranker live.
PHASE 2Week 5 — Faithfulness + RAGAS BaselineDays 29–35
  • Add Node 8 — faithfulness_check using Gemini Flash
  • Integrate RAGAS — async post-generation, push scores to Langfuse
  • Run RAGAS on 50-question eval set — establish baseline numbers
  • Add /feedback endpoint — thumbs up/down wired to Langfuse trace
  • End of week: You have RAGAS baseline. You know exactly where the system is weak.
PHASE 3Week 6+ — Data-Driven UpgradesDays 36+
  • Look at RAGAS scores — pick the lowest dimension
  • If context_precision low → semantic chunking
  • If faithfulness low on short queries → add HyDE
  • If tables/charts not retrieving → ColPali/ColQwen
  • If small-corpus queries are slow → long-context bypass with Gemini 2.5 Pro
  • Phase 3 is ongoing — let data drive, not instinct

SEC-07API Contract
API Endpoints
All endpoints require a Logto-issued access token via Authorization: Bearer <token> (except /health). The token is verified using PyJWT against the JWKS at {LOGTO_ENDPOINT}/oidc/.well-known/jwks.json (cached, 5-min lifespan) with audience LOGTO_RESOURCE. The tenant_id is always extracted server-side from the organization_id claim — never from the request body.
MethodEndpointDescriptionReturns
POST /ingest Upload a PDF. Validates, deduplicates, dispatches Celery task. Returns immediately — processing is async. {"task_id": "...", "status": "queued"}
GET /ingest/status/{task_id} Poll Celery task status. Returns: pending | processing | success | failed with error detail. {"status": "success", "chunk_count": 84}
POST /chat/stream Submit a query. Returns SSE stream: status events → token events → done event with full citations JSON. SSE stream
POST /feedback Submit thumbs up/down for a query. Body: {"trace_id": "...", "rating": 1}. Wired to Langfuse trace. Phase 2. {"status": "recorded"}
GET /documents List all active documents for the current tenant. Returns metadata only, not chunk content. [{"id": "...", "filename": "...", "version": 2}]
DELETE /documents/{doc_id} Soft-delete document — sets is_active=false on all Qdrant points and PostgreSQL parent chunks. Does not delete raw file. {"status": "deactivated"}
GET /health Health check. Returns status of all downstream services: Postgres, Qdrant, Redis. No auth required. {"status": "ok", "services": {...}}
SSE Stream Format — /chat/stream sends three event types:
event: status · data: {"node": "retrieve", "message": "Searching 84 chunks..."}
event: token · data: {"text": "According to"}
event: done · data: {"answer": "...", "citations": [...], "trace_id": "...", "faithfulness": 0.94}

from app.core.celery_app import celery_app
from app.ingestion.parser import parse_pdf_with_docling
from app.ingestion.chunker import hierarchical_chunk
from app.ingestion.embedder import embed_chunks
from app.ingestion.upserter import upsert_to_qdrant

@celery_app.task(bind=True, max_retries=3, default_retry_delay=30)
def ingest_document(self, file_path: str, tenant_id: str, document_id: str):
    try:
        # 1. Parse PDF with Docling (tables → markdown, OCR for scanned)
        pages = parse_pdf_with_docling(file_path)

        # 2. Hierarchical chunking: parent 1024t → child 256t
        parent_nodes, child_nodes = hierarchical_chunk(pages)

        # 3. Embed child chunks (dense + sparse)
        embedded_children = embed_chunks(child_nodes)

        # 4. Write parent chunks to PostgreSQL
        write_parent_chunks(parent_nodes, tenant_id, document_id)

        # 5. Upsert children to Qdrant with named vectors
        collection = f"tenant_{tenant_id}"
        upsert_to_qdrant(embedded_children, collection, document_id)

        # 6. Validate chunk count
        validate_chunk_count(collection, document_id, len(child_nodes))

    except Exception as exc:
        raise self.retry(exc=exc)

from qdrant_client.models import PointStruct, SparseVector

def build_point(chunk, dense_vec, sparse_vec) -> PointStruct:
    return PointStruct(
        id=chunk.node_id,
        vector={
            "dense": dense_vec,             # 1536-dim, openai/text-embedding-3-small via OpenRouter
            "bm25": SparseVector(           # FastEmbed sparse
                indices=sparse_vec.indices,
                values=sparse_vec.values
            )
        },
        payload={
            "tenant_id": chunk.tenant_id,
            "document_id": chunk.doc_id,
            "parent_id": chunk.parent_id,  # key for parent fetch
            "doc_type": chunk.doc_type,
            "page_num": chunk.page_num,
            "chunk_index": chunk.index,
            "is_active": True,              # versioning flag
            "text_preview": chunk.text[:200]
        }
    )

app/agent/state.py — LangGraph TypedDict

from typing import TypedDict, Literal, Annotated
from langgraph.graph.message import add_messages
from langchain_core.messages import BaseMessage

class AgentState(TypedDict):
    # ── Core query ──────────────────────────────
    query: str
    tenant_id: str
    collection: str                        # Qdrant collection name
    query_type: Literal["simple_factual", "multi_part", "procedural"]
    sub_questions: list[str]              # decomposed sub-queries
    iteration: int                         # retry counter, max 3

    # ── Retrieval ───────────────────────────────
    retrieved_chunks: list[dict]          # post-rerank top-8
    parent_chunks: list[dict]            # 1024-token context
    context_score: float                  # Gemini assessment 0-10
    context_reason: str                   # used for reformulation

    # ── Generation ──────────────────────────────
    final_answer: str
    citations: list[dict]
    faithfulness_score: float             # Phase 2

    # ── LangGraph ───────────────────────────────
    messages: Annotated[list[BaseMessage], add_messages]
app/agent/graph.py — LangGraph StateGraph wiring
Python
from langgraph.graph import StateGraph, END
from app.agent.state import AgentState
from app.agent.nodes import (
    classify_query, decompose_question, retrieve,
    rerank, parent_fetch, assess_context,
    reformulate_query, generate_answer, faithfulness_check
)

def build_graph() -> StateGraph:
    g = StateGraph(AgentState)

    # ── Add nodes ──────────────────────────────
    g.add_node("classify", classify_query)
    g.add_node("decompose", decompose_question)
    g.add_node("retrieve", retrieve)
    g.add_node("rerank", rerank)
    g.add_node("parent_fetch", parent_fetch)
    g.add_node("assess", assess_context)
    g.add_node("reformulate", reformulate_query)
    g.add_node("generate", generate_answer)
    g.add_node("faithfulness", faithfulness_check)  # Phase 2

    # ── Entry ──────────────────────────────────
    g.set_entry_point("classify")

    # ── Classify routing ───────────────────────
    g.add_conditional_edges("classify", lambda s: s["query_type"], {
        "multi_part": "decompose",
        "simple_factual": "retrieve",
        "procedural": "retrieve"
    })
    g.add_edge("decompose", "retrieve")
    g.add_edge("retrieve", "rerank")
    g.add_edge("rerank", "parent_fetch")
    g.add_edge("parent_fetch", "assess")

    # ── Assess routing ─────────────────────────
    def route_assess(state: AgentState) -> str:
        if state["context_score"] >= 7:
            return "generate"
        if state["iteration"] < 3:
            return "reformulate"
        return "generate"          # best-effort after max iterations

    g.add_conditional_edges("assess", route_assess)
    g.add_edge("reformulate", "retrieve")  # loop back
    g.add_edge("generate", "faithfulness")
    g.add_edge("faithfulness", END)

    return g.compile()

Hybrid Retrieval
app/retrieval/hybrid_search.py — Qdrant prefetch + RRF
Python
from qdrant_client.models import (
    Prefetch, Query, Fusion, Filter, FieldCondition, MatchValue
)

async def hybrid_search(
    query_text: str,
    tenant_id: str,
    collection: str,
    limit: int = 30
) -> list:

    # Embed query — dense
    dense_vec = await openai_embed(query_text)
    # Embed query — sparse
    sparse_vec = bm25_embed(query_text)

    # Qdrant tenant + version filter
    tenant_filter = Filter(must=[
        FieldCondition(key="tenant_id", match=MatchValue(value=tenant_id)),
        FieldCondition(key="is_active", match=MatchValue(value=True))
    ])

    results = qdrant_client.query_points(
        collection_name=collection,
        prefetch=[
            Prefetch(query=dense_vec, using="dense",
                     filter=tenant_filter, limit=limit),
            Prefetch(query=sparse_vec, using="bm25",
                     filter=tenant_filter, limit=limit),
        ],
        query=Query(fusion=Fusion.RRF),  # Reciprocal Rank Fusion
        limit=limit,
        with_payload=True
    )
    return results.points

import redis, json, numpy as np
from app.core.config import settings

cache_redis = redis.Redis.from_url(settings.SEMANTIC_CACHE_REDIS_URL)

async def check_cache(query: str, tenant_id: str) -> dict | None:
    query_vec = await openai_embed(query)
    cache_keys = cache_redis.keys(f"cache:{tenant_id}:*")

    for key in cache_keys:
        entry = json.loads(cache_redis.get(key))
        cached_vec = np.array(entry["embedding"])
        similarity = np.dot(query_vec, cached_vec) / (
            np.linalg.norm(query_vec) * np.linalg.norm(cached_vec)
        )
        if similarity >= settings.CACHE_SIMILARITY_THRESHOLD:  # 0.88
            return entry["response"]
    return None

def write_cache(query: str, embedding, response: dict, tenant_id: str):
    key = f"cache:{tenant_id}:{hash(query)}"
    cache_redis.setex(key, settings.CACHE_TTL_SECONDS, json.dumps({
        "embedding": embedding,
        "response": response
    }))

from transformers import AutoTokenizer, AutoModelForSequenceClassification
import torch

# Load once on startup — cached for lifetime of worker process
_tokenizer = None
_model = None

def _load_model():
    global _tokenizer, _model
    if _model is None:
        model_name = "BAAI/bge-reranker-v2-m3"
        _tokenizer = AutoTokenizer.from_pretrained(model_name)
        _model = AutoModelForSequenceClassification.from_pretrained(model_name)
        _model.eval()
    return _tokenizer, _model

def rerank_chunks(query: str, chunks: list, top_k: int = 8) -> list:
    tokenizer, model = _load_model()
    pairs = [(query, c["text_preview"]) for c in chunks]

    with torch.no_grad():
        inputs = tokenizer(pairs, padding=True, truncation=True,
                           max_length=512, return_tensors="pt")
        scores = model(**inputs).logits.squeeze(-1).tolist()

    ranked = sorted(zip(chunks, scores), key=lambda x: x[1], reverse=True)
    return [chunk for chunk, _ in ranked[:top_k]]

import json
from app.agent.state import AgentState
from app.core.config import settings
from app.core.llm import get_async_openai_client   # OpenRouter-backed

FAITHFULNESS_PROMPT = """
You are a faithfulness verifier. Given an answer and source chunks,
check if every factual claim in the answer is supported by the sources.

Answer: {answer}

Source chunks:
{sources}

Respond ONLY with valid JSON:
{{"faithfulness_score": 0.0-1.0, "unsupported_claims": ["..."]}}
"""

async def faithfulness_check(state: AgentState) -> AgentState:
    sources_text = "\n---\n".join(
        c["text"] for c in state["parent_chunks"]
    )
    prompt = FAITHFULNESS_PROMPT.format(
        answer=state["final_answer"],
        sources=sources_text
    )

    client = get_async_openai_client()
    response = await client.chat.completions.create(
        model="google/gemini-2.5-flash",           # or settings.LLM_MODEL_ASSESS
        messages=[{"role": "user", "content": prompt}],
        temperature=0,
        response_format={"type": "json_object"},
    )
    result = json.loads(response.choices[0].message.content)
    score = result["faithfulness_score"]

    # Regenerate if score is too low
    if score < 0.85:
        # Trigger stricter regeneration (add unsupported claims to prompt)
        state = await regenerate_with_constraints(state, result["unsupported_claims"])

    return {**state, "faithfulness_score": score}

from langfuse.decorators import observe, langfuse_context
from app.agent.state import AgentState
from app.retrieval.hybrid_search import hybrid_search

@observe(name="retrieve_node")  # every call creates a span in Langfuse
async def retrieve(state: AgentState) -> AgentState:
    query = state["query"]
    results = await hybrid_search(
        query_text=query,
        tenant_id=state["tenant_id"],
        collection=state["collection"]
    )

    # Log retrieval metadata to current Langfuse span
    langfuse_context.update_current_observation(
        input={"query": query},
        output={"result_count": len(results)},
        metadata={
            "top_scores": [r.score for r in results[:5]],
            "collection": state["collection"]
        }
    )
    return {**state, "retrieved_chunks": [r.payload for r in results]}
Apply @observe() to every node: classify, decompose, retrieve, rerank, parent_fetch, assess, generate, faithfulness. Each becomes a span in the Langfuse trace tree. The full trace shows you exactly which node took how long and what data flowed through it. This is the single most valuable debugging tool in the entire system.
