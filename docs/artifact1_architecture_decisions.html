ADR-01Project Context
What We're Building
A full-complexity, personal Agentic RAG system. General-purpose (not domain-locked). Fully self-hosted via Docker. Portfolio-grade architecture — built to learn and showcase, not to cut corners.
What this IS
✓General-purpose document Q&A — works on any PDF domain
✓Full agentic loop with self-correction (CRAG pattern)
✓Hybrid retrieval: dense + sparse with RRF fusion
✓Fully self-hosted — no AWS, no managed cloud queues
✓Observability-first with Langfuse self-hosted
✓Multi-tenant ready (collection-per-tenant in Qdrant)
✓Portfolio-grade complexity — no shortcut taken on core arch
What we DROPPED (and why)
✗AWS SQS — replaced by Redis Streams (already in stack)
✗AWS Lambda — replaced by Celery worker (Docker container)
✗Kafka + Zookeeper — overkill for one-user volume, adds complexity
✗Cohere Rerank API — replaced by local BGE model (free, faster)
✗Keyword-based query classifier — replaced by LLM classifier
✗LlamaIndex PDFReader (basic) — replaced by Docling
✗BFSI-only scope — now general-purpose

ADR-06Agent Design
AgentState Schema
The Pydantic model that flows through every LangGraph node. Every field has a purpose. This is the single source of truth for what the agent knows at any point in the pipeline.
AgentState — Pydantic Model (LangGraph TypedDict)
Core Query Fields
query: str — original user query
tenant_id: str — from Logto organization_id claim, sets Qdrant collection
collection: str — Qdrant collection name
query_type: Literal[...] — simple_factual | multi_part | procedural
sub_questions: list[str] — decomposed sub-queries (complex path)
iteration: int — current retry count, max 3
Retrieval & Generation Fields
retrieved_chunks: list[Chunk] — post-rerank top-8
parent_chunks: list[ParentChunk] — 1024-token context
context_score: float — Gemini Flash assessment 0-10
context_reason: str — assessor explanation for reformulation
final_answer: str — GPT-4o generated answer
citations: list[Citation] — [Source: doc, Page X] structured
faithfulness_score: float — NEW: post-generation verification
messages: list[BaseMessage] — LangGraph message history
Routing Logic (conditional edges): classify_query result → routes to retrieve OR decompose. assess_context score + iteration → routes to generate, reformulate_query, or fallback. Reformulate uses context_reason to rewrite the query intelligently before looping back to retrieve. Max iteration = 3 hard stop.

ADR-08Formal Decisions
Key Decision Records
The decisions that had real alternatives worth recording. Each ADR captures the context, the options considered, and the final call.
ADR-001 · QUEUE TECHNOLOGY
Async ingestion queue: Redis Streams over Kafka
Accepted
Redis Streams is the async queue for PDF ingestion tasks, replacing SQS. Kafka was evaluated and rejected.
Context
Personal project, Docker-only, single user. Original used AWS SQS. Needed a self-hosted async queue. Both Kafka and Redis Streams were considered after SQS was dropped.
Options Considered
Kafka: powerful, distributed, correct for high-volume. Requires Zookeeper. Extra 2 containers. Overkill for 1-user ingestion volume.

Redis Streams: Redis already in stack. Consumer groups. Persistence. 0 extra containers.
Decision & Consequence
Redis Streams wins. Celery uses Redis as broker — same Redis instance, DB 0. If volume ever needs Kafka: Celery broker swap is a one-line config change. No app code changes needed.
ADR-002 · RERANKER
Local BGE reranker over Cohere API
Accepted
BGE-Reranker-v2-M3 runs in-process in the Celery worker. Cohere API dependency removed entirely.
Context
Cohere rerank was 210ms per query (11.7% of total latency), costs ~$0.0005/query, and adds an external API dependency. Local alternatives are now production-mature in 2026.
Options Considered
Keep Cohere: simplest, highest quality historically. But 210ms and paid API.

BGE-v2-M3: ~568MB model, 15-25ms, free, multilingual, within 2% quality of Cohere on BEIR.

Jina reranker-v2: Apache licensed, similar quality.
Decision & Consequence
BGE-v2-M3 chosen. Loads on worker startup (~3s cold start, then warm). Saves 190ms/query, eliminates API cost and dependency. If quality proves insufficient: Cohere is a one-line swap-back.
ADR-003 · QUERY ROUTING
LLM classifier over keyword-based routing
Accepted
Gemini Flash at temperature 0 with structured JSON output replaces the keyword list for query classification.
Context
Original keyword list ["compare", "difference", " and ", "also", " vs "] caused systematic misclassification — " and " fires on ~50% of all English queries regardless of complexity.
Options Considered
Improve keywords: add more rules, longer patterns. Still brittle, doesn't scale.

Fine-tune classifier: train a small text classifier on labeled queries. Better quality, requires labeled data.

LLM (Gemini Flash): immediate, no data needed, ~$0.00005/call.
Decision & Consequence
LLM classifier chosen for speed of implementation and quality. Cache classifier results in Redis (query embedding key, 24h TTL) to avoid repeated LLM calls on identical queries. Adds ~80ms to cold-path only.
ADR-004 · CHUNKING STRATEGY
Fixed-size chunking kept for Phase 1 (semantic chunking deferred)
Phase 1 Only
HierarchicalNodeParser with 1024/256 fixed token sizes kept for Phase 1. Semantic chunking (SemanticSplitterNodeParser) scheduled for Phase 2 after eval data collected.
Context
Fixed-size chunking violates semantic boundaries — a sentence or table row can be split mid-content. Semantic chunking uses embeddings to find natural break points.
Why Deferred
Semantic chunking requires re-ingesting all documents when changed. Without an eval set, we can't measure whether it actually improves retrieval quality. Need baseline first.
Trigger to Upgrade
When RAGAS context_precision score is consistently below 0.75 on your eval set, switch to SemanticSplitterNodeParser and re-ingest. The pipeline supports hot-swap without other changes.
What we deliberately kept simple (scope decisions for personal project): No rate limiting per tenant (single user), no per-tenant SQS priority queues (single Celery queue is fine), no post-generation NLI verification (Gemini Flash faithfulness check is sufficient without regulatory stakes), no per-tenant Logto resource provisioning automation (manual via Logto Console for now). These aren't gaps — they're correct scope choices for a personal project that retains full portfolio-grade architecture.
